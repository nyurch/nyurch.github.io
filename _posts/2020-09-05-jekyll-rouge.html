На майбутнє, щоб не забути. Можна перейти для відображення програмного коду від конструкцій типу <xmp>...</xmp>, <code>...</code>, <code><code>...</code></code> до використання Rouge, що вже вбудовано в Jekyll 3. Плюси очевидні, незручності(мабуть екранування спецсимволів) почнуть вилазити в процесі. {% rаw %}...{% endrаw %}.

  1. Перевіряємо наявність встановлених Rouge та Kramdown gem list --local
  2. У файл config.yml додаємо наступне {% highlight yml linenos %}markdown: kramdown kramdown: input: GFM syntax_highlighter: rouge{% endhighlight %}
  3. Дивимось доступні на даний момент стилі та створюємо css-файл. rougify help style ... rougify style pastie > css/rouge.css
  4. Підключаємо стиль у основний шаблон _layouts/default.html {% highlight html linenos %}{% endhighlight %}
  5. Дивимося список підтримуваних мов та аліаси для них. Можна закинути у файл, щоб було під рукою. rougify list > rougify.list

Використання наступне - необхідний обривок коду обгортаємо тегами {% raw %}{% highlight МОВА ПАРАМЕТРИ %}...{% endhighlight %}{% endraw %}. Всі вищепоказані вставки у текстові файли уже показані за цим принципом.
